Chart and table standards

Projection · scatter · order book · dense grid
Projection and fan chartsforecast cut · range bands
Use one vertical forecast cut line and label the range table below the chart.
Keep percentile bands low-opacity; the median or observed path stays visually dominant.
Projection tables use mono values with signed deltas and explicit time horizons.
Scatter and regressioncloud · selected mark · fit
Use a muted cloud, one selected mark, and a single fit line. Avoid color-only category encoding.
Show sample count, beta, correlation, and selected point identifiers in the chart header.
Selected points should drive the row-detail panel or entity summary.
Order-book depthbid · ask · spread
Depth charts use mirrored bid/ask fills, a visible spread marker, and mono price labels.
Pair the chart with a dense price-level table. Cumulative quantity is right-aligned.
Selected price levels should remain stable when refresh updates arrive.

Headers are sticky and mono. Numeric columns are right-aligned with tabular figures.
Selected rows must drive a visible detail panel, not just a highlight.
Group columns only when it clarifies workflow scope such as price, settlement, or evidence.
Status window
Use for run status, audit trails, packet generation, queue processing, or workflow summaries. It belongs beside or below evidence and must include owner, timestamp, and next action when available.
Row detail panel
Use for selected trades, holdings, reconciliation breaks, provider rows, and report items. Keep canonical IDs mono and preserve the selected-row link back to the table.
