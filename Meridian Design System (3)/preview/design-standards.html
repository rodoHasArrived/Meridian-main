Rule 01 / 08
Surface model & radii

Use exactly 5 surface tiers. Panels are never more than 10px radius — the tightest allowed is 3px for tags and labels. Never exceed --radius-xl (10px).

Do Use the tier that matches depth — deeper element gets a lighter background.

Don't Mix large radius (>12px) with dense data surfaces.

Rule 02 / 08
Cyan normalization

Canonical primary: #2AB2D4. WPF uses #20D3F7 and #2D9CDB in specific slots — these are not interchangeable.

Do Use --cyan-primary for all default interactive tints. Use --cyan-focus for keyboard focus rings only.

Don't Use raw hex in components. Don't use #20D3F7 as a general primary — it's too bright.

Rule 03 / 08
Workspace differentiation

Each workspace gets one accent color used for nav inset, active badge, and icon tint only. It does not change the surface palette.

Do Apply the workspace accent to the active nav item inset (2–3px left border) and the status badge.

Don't Paint entire panels in the workspace accent. Don't use amber in Research or cyan in Governance.

Rule 04 / 08
Dense data readability

All data — prices, counts, IDs, timestamps — is IBM Plex Mono, 12px, tabular figures, right-aligned where numeric. Column headers are 10px mono, uppercase, 0.12em tracked.

Do Right-align all numeric columns. Use font-variant-numeric: tabular-nums everywhere numbers appear.

Don't Mix proportional and tabular numerals in the same column. Don't use sans-serif for price display.

Rule 05 / 08
Semantic state patterns

Every state uses the same three-token structure: -fg for text/icon color, -bd for border, -bg for fill. Never invent a new color — express a new state as a variant of an existing tone.

States: healthy · warn · danger · paper · research · pending · muted

Do Use .state-{name} utility classes for badges and banners.

Don't Write inline hex for state colors. Don't use color: green.

● Healthy ● Observe ● Critical ● Paper ● Research ● Pending ● Inactive
Structure: color: var(--state-{x}-fg) · background: var(--state-{x}-bg) · border: 1px solid var(--state-{x}-bd)
Rule 06 / 08
Command palette polish

The command palette is the primary keyboard surface. It uses --border-hi (#2A4566) border, --shadow-float elevation, and the focus ring token on its input.

Results use a 4px left inset on selection, no full background change. Keyboard shortcuts are mono, 10px, in a kbd element.

Do Post slideIndexChanged and hook keyboard nav to Ctrl+K.

Don't Show more than 6 results without section headers. Don't use hover-only selection.

Rule 07 / 08
Replace legacy terminal cues

Meridian is not a Bloomberg terminal clone. Avoid blinking cursors in non-interactive surfaces, all-caps body text, and green-on-black color schemes.

Do Use mono type for data. Use the full Meridian palette. Use uppercase only for eyebrow labels and column headers.

Don't Use color: #00FF00, background: black, blinking elements, or scrolling tickers as decoration. These read as parody.

Rule 08 / 08
Chart-specific standards

All financial charts share one set of tokens. Grid lines are --chart-grid (very dark). Bull/bear candles use --chart-up and --chart-dn. Overlays follow a strict stack: MA(20) cyan → MA(50) amber → benchmark blue → Bollinger transparent fill.

Do Show price labels in cyan boxes at the right axis. Use dashed lines for projections and targets.

Don't Use red/green as the only signal — always pair with numeric value. Never use filled circles for points on a line chart — use open circles r=4, stroke=color, fill=bg.
